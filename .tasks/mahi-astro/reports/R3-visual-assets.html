Executive Summary

R3 researches and documents every visual asset needed for the Believe Astrology website: hero images, service page imagery, zodiac/chakra/service icon sets, decorative SVGs, social media images, and the full Astro image optimization pipeline.

Custom SVG Strategy

Why custom SVGs over raster images for icons?

  • Resolution-independent — crisp on all screens including Retina
  • Tiny file size — 12 zodiac SVGs total ~15KB vs ~500KB for PNGs
  • Themeable — CSS can change colors for dark/light mode
  • Animatable — subtle hover effects without JavaScript
  • Brand-consistent — exact stroke width and color across all icons

SVG Customization Rules

PropertyZodiac IconsService IconsChakra IconsUI Icons
ViewBox0 0 48 480 0 64 640 0 48 480 0 24 24
Stroke Width2px2px2px2px
Default Color#C9A84C (Gold)#6F4278 (Purple)Per-chakra color#64748b (Slate)
Hover Color#FF9933 (Saffron)#C9A84C (Gold)Brighter variant#6F4278 (Purple)
Fillnone (outline)none (outline)Semi-transparentnone (outline)

Workflow

  1. Download base SVGs from free sources (SVG Repo, Hexmos, Flaticon free tier)
  2. Open in text editor — SVGs are just XML
  3. Replace hard-coded colors with currentColor for CSS theming
  4. Standardize viewBox, stroke-width, and remove unnecessary metadata
  5. Save to public/images/icons/ organized by category
  6. Create an Astro component that renders inline SVG with props for size/color

Astro Image Configuration

Astro <Image /> and <Picture /> Components

Astro 5.x provides two built-in components for automatic image optimization:

<Image /> — Single Format

Converts to WebP by default. Use for most images.

--- import { Image } from 'astro:assets'; import heroImg from '../assets/images/hero-cosmic.jpg'; --- <Image src={heroImg} alt="Cosmic night sky" width={1920} height={800} layout="responsive" loading="eager" fetchpriority="high" />

<Picture /> — Multiple Formats

Generates AVIF + WebP + fallback JPG. Use for hero images where every byte matters.

--- import { Picture } from 'astro:assets'; import heroImg from '../assets/images/hero-cosmic.jpg'; --- <Picture src={heroImg} formats={['avif', 'webp']} alt="Cosmic night sky — Believe Astrology" width={1920} height={800} layout="full-width" loading="eager" />

Global Configuration (astro.config.mjs)

// astro.config.mjs export default defineConfig({ image: { // Enable responsive images globally layout: "responsive", // Allowed remote image domains (if needed) domains: ["images.unsplash.com"], }, });

Key Rules

  • Images must be imported in .astro files (not URL strings) for optimization to work
  • Store source images in src/assets/images/ (processed by Astro) — NOT in public/
  • SVG icons go in public/images/icons/ (served as-is, no processing needed)
  • Always set width and height to prevent CLS
  • Use loading="eager" only for above-the-fold hero images (1-2 per page)
